# PR workflow

Draft scheduled for a later v1.3.x release.

This page will describe a small-batch PR workflow suitable for low-throughput projects: 1-3 goals per release, feature branches to main via PR, one issue at a time, short-lived branches, and release versioning on each merge.

For the current source of this approach, see the originating project's wiki page on how we work.
